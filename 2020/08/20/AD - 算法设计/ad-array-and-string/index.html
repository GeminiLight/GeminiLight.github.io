【数据结构】数组和字符串

发表于 2020-08-20 分类于 AD - 算法设计 本文字数： 1.1k 阅读时长 ≈ 2 分钟

数据结构之数组和字符串

数据简介

集合、列表和数组

集合

由一个或多个确定的元素所构成的整体

  • 集合里的元素类型不一定相同
  • 集合里的元素没有顺序

很多数据结构，就是在集合的基础上添加了一些规则形成的。

列表

又称线性列表，一种数据项构成的有限序列，即按照一定的线性顺序，排列而成的数据项的集合。

  • 它具有顺序，
  • 长度是可变的（可进行增删）

栈和队列则是两种特殊类型的列表

数组

数组是列表的实现方式之一，且具有自己的特征。

不同的语言也有一定差别：

  • C++ 和 Java 中，数组中的元素类型必须保持一致
  • Python 中则可以不同。Python 中的数组叫做 list，具有更多的高级功能

区分列表和数组：

  • index（索引）

    • 数组会用一些名为index的数字来标识每项数据在数组中的位置
      • 索引一般是从 0 算起的，可根据索引快速访问数组中的元素
    • 列表中没有索引

  • 内存存储方式

  • 数组中的元素在内存中是连续存储的，且每个元素占用相同大小的内存

  • 列表中的元素在内存中可能彼此相邻，也可能不相邻。比如链表的元素在内存中不一定是连续的。

数组操作

读取元素

通过访问索引的方式来读取的，索引一般从0开始

在计算机中，内存可以看成一些已经排列好的格子，每个格子对应一个内存地址。一般情况下，数据会分散地存储在不同的格子中。

对于数组，计算机会在内存中为其申请一段 连续 的空间，并且会记下索引为 0 处的内存地址。以数组 ["C", "O", "D", "E", "R"] 为例，它的各元素对应的索引及内存地址如下图所示

计算内存地址这个过程是很快的，而我们一旦知道了内存地址就可以立即访问到该元素，因此它的时间复杂度是常数级别，为 O(1)

查找元素

由于我们只保存了索引为 0 处的内存地址，因此在查找元素时，只需从数组开头逐步向后查找。

  • 如果数组中的某个元素为目标元素，
  • 则停止查找；否则继续搜索直到到达数组的末尾

最坏情况下，搜索的元素为 "R"，或者数组中不包含目标元素时，我们需要查找 n 次，n 为数组的长度，因此查找元素的时间复杂度为 O(N)
